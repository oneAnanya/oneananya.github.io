Implementing Normalization Using MySQL

Normalization is a process that organizes data into multiple related tables to eliminate redundancy and ensure data integrity.

Unnormalized Table (UNF)

The unnormalized table contains redundant and multivalued attributes. The goal of normalization is to break it down into smaller, structured tables.

OrderID CustomerName Product Quantity Supplier
1 Alice Laptop, Mouse 1, 2 HP, Logitech
2 Bob Keyboard 1 Logitech

1. First Normal Form (1NF)

A table is in 1NF if:

  • Each column contains atomic values.
  • Each row has a unique identifier (Primary Key).

Operation: Remove duplicate or multivalued attributes by creating a separate table for them.

OrderID CustomerName Product Quantity Supplier
1 Alice Laptop 1 HP
1 Alice Mouse 2 Logitech

2. Second Normal Form (2NF)

A table is in 2NF if:

  • It is in 1NF.
  • All non-key attributes depend on the whole primary key.

Operation: Remove partial dependencies by splitting the table into separate tables.

3. Third Normal Form (3NF)

A table is in 3NF if:

  • It is in 2NF.
  • It has no transitive dependencies.

4. Boyce-Codd Normal Form (BCNF)

A table is in BCNF if:

  • It is in 3NF.
  • For every functional dependency X → Y, X should be a superkey.

Benefits of Normalization

  • Eliminates Data Redundancy
  • Ensures Data Integrity
  • Improves Query Performance
  • Maintains Data Consistency
