ACID Properties in a Relational Database

ACID properties (Atomicity, Consistency, Isolation, Durability) ensure reliable transactions in a relational database. Here’s how these properties apply to our Institute database with the 5 relational tables i.e., Students, Courses, Student-Hobby, Course-Teacher, and Student-Teacher.

1. Atomicity

Atomicity ensures that all parts of a transaction are completed successfully or none are. For example, when enrolling a new student:

  • Operation: Insert data into the Students, Courses, and Student-Teacher tables.
  • Scenario: If adding a student to Students succeeds but fails in Courses, the transaction should roll back, leaving the database unchanged.

2. Consistency

Consistency ensures the database transitions from one valid state to another. For example, if we have a constraint that every student must have a unique StudentID, it ensures no duplicates:

  • Scenario: Trying to insert a duplicate StudentID would violate the consistency rule, so the transaction would fail, preserving database integrity.

3. Isolation

Isolation ensures that transactions are executed independently without interference. For example, two students enrolling simultaneously should not affect each other’s data:

  • Scenario: If two transactions try to enroll students simultaneously, isolation ensures each transaction operates as if it’s the only one in the system.

4. Durability

Durability ensures that once a transaction is committed, it remains in the database even in case of a system failure. For example, once a student’s enrollment is confirmed:

  • Scenario: If a power failure occurs right after a commit, the data remains safe.
