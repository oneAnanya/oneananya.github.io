Types of Joins in SQL

Joins in SQL are used to combine rows from two or more tables based on a related column. Here are the common types of joins with examples:

1. INNER JOIN

The INNER JOIN keyword selects records that have matching values in both tables.

Scenario: You are working in a university and need to list students along with the courses they are currently enrolled in.

To find records where there is a match between the Students and Courses tables.

Students Table:

StudentID Name
1 Ananya
2 Ankita
3 Archita

Courses Table:

StudentID Course
1 PGDCA
2 Full Stack

SQL:

SELECT Students.Name, Courses.Course
FROM Students
INNER JOIN Courses ON Students.StudentID = Courses.StudentID;

Output:

Name Course
Ananya PGDCA
Ankita Full Stack

2. LEFT JOIN (LEFT OUTER JOIN)

The LEFT JOIN keyword returns all records from the left table, and the matched records from the right table. The result is NULL from the right side if there is no match.

Scenario: You are analyzing student enrollment and want to list all students, including those who might not be enrolled in any courses.

To include all records from the left table Students and matched records from the right table Courses. If there are students without any courses, they will still be listed with NULL for the course.

SQL:

SELECT Students.Name, Courses.Course
FROM Students
LEFT JOIN Courses ON Students.StudentID = Courses.StudentID;

Output:

Name Course
Ananya PGDCA
Ankita Full Stack
Archita NULL

3. RIGHT JOIN (RIGHT OUTER JOIN)

The RIGHT JOIN keyword returns all records from the right table, and the matched records from the left table. The result is NULL from the left side when there is no match.

SQL:

SELECT Students.Name, Courses.Course
FROM Students
RIGHT JOIN Courses ON Students.StudentID = Courses.StudentID;

Output:

Name Course
Ananya PGDCA
Ankita Full Stack
NULL BI

4. FULL JOIN (FULL OUTER JOIN)

The FULL JOIN keyword returns all records when there is a match in either left or right table. It returns NULL for non-matching rows from both tables.

SQL:

SELECT Students.Name, Courses.Course
FROM Students
FULL JOIN Courses ON Students.StudentID = Courses.StudentID;

Output:

Name Course
Ananya PGDCA
Ankita Full Stack
Archita NULL
NULL BI

5. CROSS JOIN

The CROSS JOIN keyword returns the Cartesian product of two tables, combining each row from the first table with every row from the second table.

SQL:

SELECT Students.Name, Courses.Course
FROM Students
CROSS JOIN Courses;

Output:

Name Course
Ananya PGDCA
Ananya Full Stack
Ananya BI
Ankita PGDCA
Ankita Full Stack
Ankita BI
Archita PGDCA
Archita Full Stack
Archita BI
